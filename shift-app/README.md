# シフト管理アプリ

アルバイトスタッフの出勤希望をカレンダー上で入力し、管理者がシフトをアサインできる
Webアプリです。複数の端末(スタッフのスマホ・管理者のPCなど)から同じデータを
共有できるよう、データベースに Firebase Firestore を使っています。

サーバーのプログラムは書いていません。静的なHTML/JS/CSSファイルだけで動きます。

## できること

- スタッフ: 自分の名前を選んで、カレンダー上で出勤希望日をON/OFF切り替え
- 管理者: カレンダー上で日付ごとの希望者一覧を確認し、アサイン(時間指定つき)を保存
- スタッフの登録・削除

## 事前準備(初回のみ)

セットアップは大きく2ステップです。

1. Firebase プロジェクトを作る(データの保存先)
2. どこかにファイルを公開してURLを発行する(スタッフ・管理者がアクセスするため)

所要時間は15分程度です。

---

## ステップ1: Firebaseプロジェクトを作る

1. https://console.firebase.google.com/ にアクセスし、Googleアカウントでログイン
2. 「プロジェクトを作成」→ 好きなプロジェクト名(例: `shift-app`)を入力 → 作成
   (Googleアナリティクスは無効のままでOKです)

### 1-1. Firestore データベースを有効化

1. 左メニュー「構築」→「Firestore Database」→「データベースの作成」
2. ロケーションは `asia-northeast1`(東京)などお好みで選択
3. 「本番環境モードで開始」を選択(セキュリティルールは後で設定します)

### 1-2. セキュリティルールを設定

「Firestore Database」→「ルール」タブを開き、以下の内容に置き換えて「公開」します。
これは「ログインしている人(匿名ログイン含む)だけが読み書きできる」設定です。

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

> 注意: これはURLを知っている人なら誰でも操作できる、簡易的な保護です。
> 社内の少人数チームで使う想定の設定です。厳密な認証(パスワードログインなど)が
> 必要な場合は別途ご相談ください。

### 1-3. 匿名認証を有効化

1. 左メニュー「構築」→「Authentication」→「始める」
2. 「Sign-in method」タブ →「匿名」を選択 → 有効にする → 保存

### 1-4. ウェブアプリを登録し、設定値を取得

1. プロジェクトのトップ画面(歯車アイコン →「プロジェクトの設定」)を開く
2. 「マイアプリ」→ `</>`(ウェブ)アイコンをクリック
3. アプリのニックネームを適当に入力(例: `shift-web`)→「アプリを登録」
4. 表示される `firebaseConfig` の値をコピー

```js
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "shift-app-xxxx.firebaseapp.com",
  projectId: "shift-app-xxxx",
  storageBucket: "shift-app-xxxx.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:xxxxxxxx"
};
```

5. このフォルダの `firebase-config.js` を開き、`YOUR_API_KEY` などの部分を
   コピーした値にすべて置き換えて保存します。
6. 同じファイルの `ADMIN_PIN`(初期値 `1234`)も、管理者用の合言葉として
   好きな値に変更しておくことをおすすめします。

## ステップ2: ファイルを公開してURLを発行する

一番簡単なのは **Netlify Drop**(アカウント登録不要)です。

1. https://app.netlify.com/drop を開く
2. `shift-app` フォルダ(この README がある フォルダ)ごと、ブラウザの画面上に
   ドラッグ&ドロップする
3. 数秒でアップロードが完了し、`https://xxxxx.netlify.app` のようなURLが発行される
4. このURLをスタッフ・管理者に共有すれば、それぞれのスマホ・PCからアクセスできます

> 発行されたURLは一時的なものです。恒久的に使いたい場合は、Netlifyに無料アカウントで
> ログインしてサイトを「Claim(引き取り)」してください(Drop画面に案内が出ます)。

他にも Firebase Hosting、Vercel、GitHub Pages などでも公開できます。

---

## 使い方

- **スタッフ用タブ**: 自分の名前を選択 →カレンダーの日付をクリックして出勤希望を
  ON/OFF(緑色 = 希望あり)
- **管理者用タブ**(PIN入力が必要): カレンダーの日付をクリック →その日の
  希望者一覧が開くので、アサインするスタッフにチェックを入れ、時間を指定して保存
- **スタッフ管理タブ**(PIN入力が必要): スタッフの追加・削除

管理者用PINの初期値は `1234` です。`firebase-config.js` の `ADMIN_PIN` で変更できます。

## ファイル構成

```
shift-app/
├── index.html         画面のHTML
├── style.css           スタイル
├── app.js               アプリのロジック(Firestore連携)
├── firebase-config.js   ★あなたのFirebase設定値を入れるファイル
└── README.md            この手順書
```

## うまく動かないときは

- 画面に「初期設定が必要です」と出る → `firebase-config.js` の値が正しいか確認
- スタッフ一覧やカレンダーが反応しない → Firestoreの「匿名認証」が有効になっているか、
  セキュリティルールが正しく公開されているか確認
- ブラウザの開発者ツール(F12など)の「コンソール」にエラーが出ていないか確認
